Create src/lib/scrapers/career-page.ts:

CareerPageCrawler class implementing ScraperAdapter with platform = "career-page".

The discover method needs a userId parameter to fetch career page URLs. Since ScraperAdapter.discover takes SearchParams only, extend the adapter: add an optional `userId?: string` to SearchParams in types.ts, OR pass userId via constructor. Use constructor approach: `new CareerPageCrawler(userId)`.

Key methods:

1. `discover(params)`:
   - Fetch all CareerPageUrl entries for this.userId from Prisma
   - For each URL: fetchAndExtract(url, label)
   - Collect all DiscoveredJob results
   - Filter through isInternshipRole + isUSLocation
   - randomDelay(2000, 4000) between URLs

2. `fetchPage(url)`:
   - First try HTTP fetch with getRandomUserAgent()
   - Check if response is minimal (isMinimalContent) -- less than 200 chars meaningful text, or body contains `<div id="root"></div>` / `<div id="app"></div>` with no other content
   - If minimal: retry with Playwright (createStealthBrowser, navigate, wait for networkidle, get page.content())
   - Return HTML string

3. `extractMainContent(html)`:
   - Strip `<script>`, `<style>`, `<nav>`, `<footer>`, `<header>` tags and their content
   - Remove all remaining HTML tags
   - Collapse whitespace
   - Cap at 30,000 characters

4. `identifyJobLinks(html, baseUrl)`:
   - Find all href values in the HTML
   - Filter to links containing: /job/, /jobs/, /position/, /career/, /opening/, /apply/, or URLs with numeric ID segments
   - Resolve relative URLs against baseUrl
   - Deduplicate and cap at 20 links

5. `extractJobsFromPage(text, companyName)`:
   - Call Vercel AI SDK `generateObject` with anthropic("claude-sonnet-4-20250514") model
   - Zod schema for output: `{ jobs: z.array(z.object({ title: z.string(), url: z.string().nullable(), location: z.string().nullable(), department: z.string().nullable(), datePosted: z.string().nullable() })) }`
   - Prompt: "Extract all job/internship listings from this career page content. For each job, extract the title, URL (full or relative), location, department, and date posted if visible. Company: {companyName}\n\nPage content:\n{text}"
   - Map results to DiscoveredJob format

6. `fetchAndExtract(url, label)`:
   - Fetch the main career page
   - Extract jobs from the main page
   - Identify job-like links on the page
   - For each job link (1 level deep): fetch, extract text, extract jobs
   - Cap total pages per URL at 20 to prevent runaway crawling
   - Combine all extracted jobs

Env var: ANTHROPIC_API_KEY (used by @ai-sdk/anthropic automatically).
