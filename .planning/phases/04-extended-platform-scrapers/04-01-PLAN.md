Create src/lib/scrapers/workday.ts:
- `WorkdayScraper` class implementing `ScraperAdapter` with `platform = "workday"`
- `discover(params)` iterates WORKDAY_COMPANIES, calls `fetchCompanyJobs` for each
- `fetchCompanyJobs(company)` POSTs to `https://{domain}/wday/cxs/{slug}/{site}/jobs` with JSON body `{ appliedFacets: {}, limit: 20, offset: 0, searchText: "" }`
- Paginate: increment offset by limit, stop when offset >= total or postings array is empty
- Map each posting: title from `posting.title`, location from `posting.locationsText`, URL constructed from domain + `/en-US/job/` + posting externalPath, datePosted from `posting.postedOn`
- Filter all results through `isInternshipRole(title)` and `isUSLocation(location)`
- `randomDelay(1000, 3000)` between companies, `randomDelay(500, 1500)` between pages
- Catch per-company errors, log warning, continue to next company

BROWSER FALLBACK (per user decision -- JSON API first, fall back to stealth browser if API is blocked):
- `fetchCompanyJobsBrowser(company)` as fallback method, called when `fetchCompanyJobs` gets 403 or 429 response status
- In `fetchCompanyJobs`, detect 403/429 responses. When detected, log "API blocked for {company.name}, falling back to browser" and call `fetchCompanyJobsBrowser(company)` instead of returning empty
- `fetchCompanyJobsBrowser(company)`:
  1. Use `createStealthBrowser()` from stealth.ts to launch browser
  2. Navigate to `https://{domain}/{slug}/{site}` (the public career page URL)
  3. Wait for job listing elements to render (wait for `[data-automation-id="jobItem"]` or `.WDGD .WDP8` selectors, with 10s timeout)
  4. Extract job data from rendered HTML: title from job card heading, location from location text element, URL from link href
  5. Paginate by clicking "Show More" / next page button if present, up to 10 pages
  6. Close browser in finally block
  7. Apply same `isInternshipRole` + `isUSLocation` filtering to browser-extracted results
  8. Map extracted data to DiscoveredJob format (same as API path)
- If browser fallback also fails, log error and continue to next company (never throw)

- Query params: `keywords=intern`, `location=United+States`, `f_JT=I` (internship type), `f_TPR=r604800` (past week), `start={offset}`

  - Job URL: `href="(https://www\.linkedin\.com/jobs/view/[^"]+)"` from base-card__full-link

  - Company: content inside `<h4 class="base-search-card__subtitle">...</h4>` or `<a class="hidden-nested-link">`

- Detect blocks: if response contains `<title>Sign In</title>` or status is 429/403, stop pagination immediately and return what was collected so far
- No additional filtering needed -- LinkedIn's `f_JT=I` parameter already filters for internships. Still apply `isUSLocation` as safety net
- Wrap entire discover in try/catch, return empty array on fatal error
