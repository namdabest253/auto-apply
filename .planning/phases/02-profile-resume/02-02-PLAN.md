<task type="auto">
  <name>Task 1: Resume upload API route and file storage</name>
  <files>src/app/api/resume/upload/route.ts, src/components/nav/top-nav.tsx</files>
  <action>
1. Create `src/app/api/resume/upload/route.ts` with a POST handler:
   - Authenticate via `auth()` from `@/auth` -- return 401 if no session
   - Extract file from FormData (`formData.get("resume")`)
   - Validate file type (application/pdf or application/vnd.openxmlformats-officedocument.wordprocessingml.document) -- return 400 with message if invalid
   - Validate file size (max 5MB = 5 * 1024 * 1024) -- return 400 with message if too large
   - Convert to Buffer: `Buffer.from(await file.arrayBuffer())`
   - Save original file to disk at `uploads/resumes/{userId}.{ext}` using `writeFile` from `node:fs/promises` (create directory with `mkdir({recursive: true})`)
   - Extract text: if PDF use `pdfParse(buffer)` and get `.text`, if DOCX use `mammoth.extractRawText({buffer})` and get `.value`
   - Parse text: `parseResumeText(rawText)` from `@/lib/resume-parser`
   - Save to database using Prisma upsert pattern:
     a. Upsert Profile record (userId, resumeFileName, resumeFilePath, contact fields, otherText)
     b. Delete existing child records for this profile (education, workHistory, etc.) before creating new ones (single-resume model -- re-upload replaces everything)
     c. Create new child records from parsed data (Education[], WorkHistory[], Project[], Skill[], Certification[], VolunteerWork[], Publication[])
   - Use a Prisma transaction (`prisma.$transaction`) to ensure atomicity
   - Return 200 with `{ success: true, profileId }` on success
   - Wrap in try/catch, return 500 with `{ error: "Upload failed" }` on unexpected errors
   - Add `uploads/` to `.gitignore` if not already present

2. Add "Profile" navigation link to `src/components/nav/top-nav.tsx`:
   - Add a Link (from `next/link`) to `/profile` in the nav bar, between the brand and the user section
   - Style consistently with existing nav: `text-sm text-zinc-400 hover:text-zinc-100`

3. Verify the route compiles: `bun run build`
  </action>

  </verify>

2. Create `src/app/(dashboard)/profile/components/resume-upload.tsx` (client component):
   - Drag-and-drop zone using native HTML5 dragover/drop events
   - "Browse files" button as fallback (hidden file input triggered by label click)
   - Client-side validation: file type (PDF/DOCX only) and size (5MB max) before upload
   - Shows "Parsing resume..." during upload with disabled state
   - On success: calls `onUploadComplete()` callback (parent refreshes data)
   - On error: shows red error message
   - Accepts `currentFileName?: string` prop to show currently uploaded file name
   - Style: dashed border, centered text, zinc-700 border default, blue-500 on drag hover
   - Follow the pattern from RESEARCH.md drag-and-drop code example

3. Create `src/app/(dashboard)/profile/components/section-editor.tsx` (client component):
   - Reusable component for editing a list of entries (education, work history, projects, certifications, volunteer, publications)
   - Props: `title: string`, `entries: any[]`, `onSave: (entry) => void`, `onDelete: (id: string) => void`, `renderForm: (entry, onChange) => ReactNode`, `renderDisplay: (entry) => ReactNode`
   - Shows entries in display mode with Edit/Delete buttons
   - Edit button switches entry to form mode inline
   - "Add new" button at bottom adds a blank entry in form mode
   - Save button calls onSave, cancel returns to display mode
   - Delete button calls onDelete with confirmation via Dialog component

4. Create `src/app/(dashboard)/profile/actions.ts` with server actions:
   - All actions: authenticate via `auth()`, throw if no session, use `prisma` for DB operations, call `revalidatePath("/profile")` after mutation
   - `saveContactInfo(data)`: upsert Profile with contact fields
   - `saveEducation(data)`: upsert Education entry (create if no id, update if id provided)
   - `deleteEducation(id)`: delete Education entry
   - `saveWorkHistory(data)`: upsert WorkHistory entry
   - `deleteWorkHistory(id)`: delete WorkHistory entry
   - `saveProject(data)`: upsert Project entry
   - `deleteProject(id)`: delete Project entry
   - `saveSkills(skills: string[])`: delete all existing Skill records for profile, create new ones from array
   - `saveCertification(data)`: upsert Certification entry
   - `deleteCertification(id)`: delete Certification entry
   - `saveVolunteerWork(data)`: upsert VolunteerWork entry
   - `deleteVolunteerWork(id)`: delete VolunteerWork entry
   - `savePublication(data)`: upsert Publication entry
   - `deletePublication(id)`: delete Publication entry
   - Use Zod validators from `src/lib/validators.ts` (created in Plan 01) to validate input data

5. Create `src/app/(dashboard)/profile/components/profile-editor.tsx` (client component):
   - Receives full profile data as props (contact, education[], workHistory[], projects[], skills[], certifications[], volunteerWork[], publications[], otherText)
   - Renders a single scrollable page with clear section dividers (Separator component) between sections
   - **Contact Info section**: Editable form fields for name, email, phone, LinkedIn, website, location with Save button
   - **Education section**: Uses SectionEditor with form fields for school, degree, fieldOfStudy, startDate, endDate, gpa, coursework
   - **Work History section**: Uses SectionEditor with form fields for company, title, startDate, endDate, and a textarea for bullets (one per line, split/join on newlines)
   - **Projects section**: Uses SectionEditor with form fields for name, description, url, and TagInput for technologies
   - **Skills section**: TagInput component for adding/removing skills with a Save button
   - **Certifications section**: Uses SectionEditor with form fields for name, issuer, date
   - **Volunteer Work section**: Uses SectionEditor with form fields for organization, role, startDate, endDate, description
   - **Publications section**: Uses SectionEditor with form fields for title, venue, date, url
   - **Other/Unparsed section**: Read-only textarea showing otherText (if any content exists)
   - Each section has its own Save button calling the corresponding server action
   - Use Card components to group each section visually

   - Render page title "Profile"
   - Render ResumeUpload component (pass current file name if exists)
   - If profile exists: render ProfileEditor with all data
   - If no profile: show message "Upload a resume to get started, or add your information manually below" and render empty ProfileEditor
