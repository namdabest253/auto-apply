RED phase:

GREEN phase:
1. Create `src/lib/resume-parser.ts` exporting `parseResumeText(rawText: string): ParsedProfile`.
2. Implement section detection using regex patterns for common section headers (see RESEARCH.md SECTION_PATTERNS).
3. Split text into sections at detected headers.
4. For each section type, implement extraction logic:
   - Contact info: scan first 5 lines for email regex, phone regex, LinkedIn URL, name (first non-empty line)
   - Education: look for "University/College/School" keywords, degree patterns ("Bachelor/Master/BS/MS/BA"), date ranges ("2020-2024", "May 2024"), GPA pattern
   - Work history: look for company names (line before date range), title (often on same line), dates, indented bullets starting with - or *
   - Skills: split by commas, pipes, or newlines within the skills section
   - Projects: name on first line, description following, URL if present
   - Certifications: one per line in the certifications section
   - Volunteer: organization + role pattern similar to work history
   - Publications: one per line in publications section
5. Anything not consumed by a recognized section goes to `otherText`.
6. Run tests -- they must all PASS.
7. Commit: `feat(02-01): implement rule-based resume parser`
