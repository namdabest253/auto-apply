  <files>

  <action>
    1. Create src/auth.ts following the exact pattern from 01-RESEARCH.md Pattern 1:
       - Import NextAuth, PrismaAdapter, Credentials, Google, GitHub providers
       - Configure PrismaAdapter with prisma singleton
       - Use `session: { strategy: "jwt" }` (CRITICAL -- Credentials provider requires JWT, per Pitfall 1)
       - Credentials provider: validate with signInSchema, find user by email, verify password with verifyPassword, return user object
       - Google and GitHub providers with AUTH_GOOGLE_ID/SECRET and AUTH_GITHUB_ID/SECRET env vars
       - JWT callback: copy user.id to token.id
       - Session callback: copy token.id to session.user.id
       - Set custom pages: signIn -> "/login"
       - Export: handlers, auth, signIn, signOut

    2. Create src/app/api/auth/[...nextauth]/route.ts:
       ```typescript
       import { handlers } from "@/auth"
       export const { GET, POST } = handlers
       ```

    3. Create src/middleware.ts following Pattern 3 from 01-RESEARCH.md:
       - Import auth from "@/auth"
       - Export default auth callback that checks req.auth
       - If on auth page (/login or /register) and logged in -> redirect to "/"
       - If NOT on auth page and NOT logged in -> redirect to "/login"
       - Matcher excludes api, _next/static, _next/image, favicon.ico

    4. Create src/app/(auth)/register/actions.ts following the registration server action from 01-RESEARCH.md:
       - "use server" directive
       - Validate input with signInSchema
       - Check if user exists (return error if so)
       - Single-user check: if prisma.user.count() > 0, return { error: "Registration is closed" }
       - Hash password with hashPassword
       - Create user with prisma.user.create
       - Auto sign-in after registration with signIn("credentials", ...)
       - Handle errors gracefully, return { error: string } on failure

    5. Create src/app/(auth)/login/actions.ts:
       - "use server" directive
       - Import signIn from "@/auth"
       - Export login action that calls signIn("credentials", { email, password, redirectTo: "/" })
       - Wrap in try/catch, return { error: "Invalid credentials" } on AuthError

    IMPORTANT: Use AUTH_ prefix for all env vars (not NEXTAUTH_), per Pitfall 2.
    IMPORTANT: Use JWT session strategy, NOT database sessions, per Pitfall 1.
  </action>
