  </files>
  <action>
    1. Scaffold the project using `bun create next-app autoapply --typescript --app --tailwind --eslint --src-dir --import-alias "@/*"`. If this command creates a subdirectory, move its contents to the repo root.

    2. Install core dependencies:
       `bun add next-auth@beta @auth/prisma-adapter @prisma/client next-themes zod bcryptjs`

    3. Install dev dependencies:
       `bun add -d prisma vitest @vitejs/plugin-react vite-tsconfig-paths prettier @types/bcryptjs`

    4. Initialize shadcn/ui via `bunx shadcn@latest init` -- select default style, dark theme. This creates components.json and configures globals.css and lib/utils.ts. Ensure the cn() utility is in src/lib/utils.ts.

    5. Create docker-compose.yml with three services:
       - `app`: Build from Dockerfile, port 3000, depends on postgres and redis with healthchecks, sets DATABASE_URL and REDIS_URL env vars, mounts source as volume
       - `postgres`: postgres:16-alpine, user/pass/db all "autoapply", healthcheck via pg_isready, named volume
       - `redis`: redis:7-alpine, healthcheck via redis-cli ping, named volume
       Use the exact configuration from 01-RESEARCH.md Docker Compose section.

    6. Create Dockerfile with multi-stage build: oven/bun:1 for dependency install (bun install --frozen-lockfile), node:20-alpine for runtime (prisma generate + next dev). Use the pattern from 01-RESEARCH.md.

    7. Create .env.example with all required env vars (documented with comments):
       - DATABASE_URL=postgresql://autoapply:autoapply@localhost:5432/autoapply
       - REDIS_URL=redis://localhost:6379
       - AUTH_SECRET=change-me-generate-with-openssl-rand-base64-33
       - AUTH_GOOGLE_ID=
       - AUTH_GOOGLE_SECRET=
       - AUTH_GITHUB_ID=
       - AUTH_GITHUB_SECRET=
       - AUTH_URL=http://localhost:3000

    8. Create vitest.config.mts using the config from 01-RESEARCH.md (jsdom environment, tsconfigPaths plugin, react plugin, include src/**/*.test.{ts,tsx}).

    9. Update src/app/layout.tsx to a minimal root layout with `suppressHydrationWarning` on the html tag (per Pitfall 5 from research). Set lang="en". Dark mode class will be managed by next-themes in Plan 02.

    10. Ensure .gitignore includes node_modules, .next, .env, .env.local, postgres_data, redis_data.

    IMPORTANT: Use bun as package manager but node as runtime (per Pitfall 6). The Dockerfile uses oven/bun for install, node:20-alpine for running Next.js.

<task type="auto" tdd="true">
  <name>Task 2: Create Prisma schema, core libraries, and Wave 0 test scaffolds</name>
  <files>
    prisma/schema.prisma, src/lib/prisma.ts, src/lib/password.ts, src/lib/validators.ts,
    src/__tests__/lib/password.test.ts
  </files>
  <behavior>
    - hashPassword("test123") returns a string starting with "$2a$" or "$2b$" (bcrypt hash)
    - verifyPassword("test123", hashedResult) returns true
    - verifyPassword("wrong", hashedResult) returns false
    - hashPassword produces different hashes for same input (salt uniqueness)
  </behavior>
  <action>
    1. Initialize Prisma: `bunx prisma init --datasource-provider postgresql`

    2. Create prisma/schema.prisma with the Auth.js-compatible schema from 01-RESEARCH.md:
       - User model: id (cuid), name, email (unique), emailVerified, image, hashedPassword, accounts relation, createdAt, updatedAt
       - Account model: id (cuid), userId, type, provider, providerAccountId, refresh_token, access_token, expires_at, token_type, scope, id_token, session_state, user relation with cascade delete, @@unique([provider, providerAccountId])
       - VerificationToken model: identifier, token, expires, @@unique([identifier, token])

    3. Generate Prisma client: `bunx prisma generate`

    4. Create src/lib/prisma.ts with the globalThis singleton pattern from 01-RESEARCH.md.

    5. Create src/lib/password.ts with hashPassword and verifyPassword functions using bcryptjs with 12 salt rounds. Follow the exact pattern from 01-RESEARCH.md.

    6. Create src/lib/validators.ts with Zod schemas:
       ```typescript
       import { z } from "zod"
       export const signInSchema = z.object({
         email: z.string().email("Invalid email address"),
         password: z.string().min(8, "Password must be at least 8 characters"),
       })
       ```

    7. Write src/__tests__/lib/password.test.ts (TDD RED-GREEN):
       - Test: hashPassword returns bcrypt hash string
       - Test: verifyPassword returns true for correct password
       - Test: verifyPassword returns false for wrong password
       - Test: hashPassword produces unique hashes (salt)
       Run tests and confirm they pass.
  </action>

    <automated>cd /root/job && bunx prisma validate && bunx vitest run src/__tests__/lib/password.test.ts --reporter=verbose 2>&1 | tail -20</automated>
  </verify>
  <done>Prisma schema validates, Prisma client generates, password hash/verify tests pass, validators module exports signInSchema.</done>
