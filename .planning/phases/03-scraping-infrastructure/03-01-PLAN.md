  <name>Task 2: Indeed + Greenhouse scraper adapters with tests</name>

  <behavior>
    - Test: IndeedScraper.discover parses embedded JSON from window.mosaic.providerData and returns DiscoveredJob[]
    - Test: IndeedScraper constructs correct Indeed URL from keywords, locations, and internship job type
    - Test: IndeedScraper handles empty/missing mosaic data gracefully (returns empty array)
    - Test: IndeedScraper handles CAPTCHA/block page detection (throws or returns empty with warning)
    - Test: GreenhouseScraper.discover fetches from boards-api.greenhouse.io and transforms response to DiscoveredJob[]
    - Test: GreenhouseScraper maps board token to company name from GREENHOUSE_COMPANIES list
    - Test: GreenhouseScraper handles API error (non-200) gracefully
    - Test: GreenhouseScraper filters jobs by keywords when provided
  </behavior>
  <action>
1. Create src/lib/scrapers/indeed.ts implementing ScraperAdapter:
   - `platform = "indeed"`
   - `discover(params)`: builds Indeed URL with query params (q=keywords joined by space, l=first location, jt=internship). Uses createStealthBrowser() to launch browser. Navigates to URL with randomDelay between pages. Extracts embedded JSON via `window.mosaic.providerData["mosaic-provider-jobcards"]` regex from page content. Parses job cards from the JSON structure. Maps each to DiscoveredJob. Limits to 3-5 result pages per run (per research recommendation). Detects CAPTCHA/block pages by checking for known block indicators in HTML. Closes browser in finally block.
   - Handle errors gracefully: if blocked, log warning and return whatever was collected (partial-safe per user decision).

2. Create src/lib/scrapers/greenhouse.ts implementing ScraperAdapter:
   - `platform = "greenhouse"`
   - `discover(params)`: iterates over GREENHOUSE_COMPANIES list. For each company, fetches `https://boards-api.greenhouse.io/v1/boards/{slug}/jobs?content=true`. Parses JSON response. Maps each job to DiscoveredJob using company name from the curated list (NOT the slug). Strips HTML from job.content for descriptionText (use simple regex or built-in approach). Filters results by keywords if provided (case-insensitive title match). Adds randomDelay(500, 1500) between API calls to be respectful.
   - Handle non-200 responses: log warning, skip that company, continue to next.

3. Write tests:
   - src/lib/scrapers/indeed.test.ts: Mock rebrowser-playwright page.content() to return HTML containing embedded mosaic JSON. Verify correct DiscoveredJob[] output. Test block detection. Test URL construction.
   - src/lib/scrapers/greenhouse.test.ts: Mock global fetch to return Greenhouse API response JSON. Verify transformation to DiscoveredJob[]. Verify company name mapping. Test error handling for failed requests. Test keyword filtering.
